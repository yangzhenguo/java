1.1.11.Multipart解析器

  org.springframework.web.multipart包中的MultipartResolver是用来进行解析包含文件上传的multipart请求的。它有两个实现类，一个是基于Commons FileUpload的实现，另一个是基于Servlet3.0multipart请求的解析。   要开启对multipart处理的功能，你需要在DispatcherServlet中声明一个名字叫multipartResolver的的bean。DispatcherServlet会自动检测到它并让其处理捕获到的请求。当接收到一个content-type为multipart/form-data的POST请求时，这个解析器会解析请求内容并把HttpServletRequest包装成MultipartHttpServletRequest，这样就能访问到解析后的parts，并把他们暴露为请求参数。

Apache Commons FileUpload

  要想使用Apache Commons的FileUpload，你得配置一个名称叫multipartResolver的CommonsMultipartResolver。并且还需要在classpath中加入commons-fileupload依赖。

Servlet 3.0

  如果要使用Servlet3.0来解析multipart，需要配置servlet容器来开启此功能，如下:

  • JavaConfig, 在Servlet的registration上设置MultipartConfigElement
  • web.xml, 在资源描述符中添加<multipart-config>元素下面的示例展示了如何在Servlet的registration上添加MultipartConfigElement:
public class AppInitializer extends AbstractAnnotationConfigDispatcherServletInitializer {

    // ...

    @Override
    protected void customizeRegistration(ServletRegistration.Dynamic registration) {

        // Optionally also set maxFileSize, maxRequestSize, fileSizeThreshold
        registration.setMultipartConfig(new MultipartConfigElement("/tmp"));
    }

}

  这个Servlet3.0的配置做好后你就可以声明一个名字叫multipartResolver的StandardServletMultipartResolver的bean
